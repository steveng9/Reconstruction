│                                                                                                                                             │
│ Two new analytical features:                                                                                                                │

│                                                                                                                                             │
│ ---                                                                                                                                         │
│ Feature 1: Row-Level RA Scores                                                                                                              │
│                                                                                                                                             │
│ 1a. scoring.py — three new functions                                                                                                        │
│                                                                                                                                             │
│ calculate_row_level_scores(original, reconstructed, hidden_features, dataset_type)                                                          │
│ Dispatcher → calls categorical or continuous variant. Returns DataFrame(n_rows × n_score_cols).                                             │
│                                                                                                                                             │
│ _row_level_categorical(original, reconstructed, hidden_features)                                                                            │
│ - Per-cell column: binary 0/1 correctness for each hidden feature (RA_row_{feat})                                                           │
│ - RA_row_mean: rarity-weighted row score:                                                                                                   │
│ score_i = Σ_j [correct_ij × rarity(val_ij)] / Σ_j rarity(val_ij)                                                                            │
│ where rarity(val) = total_records / count(val).                                                                                             │
│ Down-weights rows with only common values; up-weights rows where rare values were correctly recovered.                                      │
│                                                                                                                                             │
│ _row_level_continuous(original, reconstructed, hidden_features)                                                                             │
│ - Per-cell column: normalized absolute error |real - pred| / range ∈ [0,1] (lower = better)                                                 │
│ - RA_row_mean: mean across features                                                                                                         │
│                                                                                                                                             │
│ compute_outlier_scores(df, qi_cols, cat_cols, num_cols, method='isolation_forest', percentile=90)                                           │
│ - Operates on QI columns only (public feature space = what an adversary observes; uniqueness in QI space is the relevant privacy risk)      │
│ - method='isolation_forest': OrdinalEncoder on cat cols + StandardScaler on num cols → IsolationForest. Score = -score_samples() (higher =  │
│ more outlier-like).                                                                                                                         │
│ - method='gower_knn': Leave-one-out Gower distance to nearest neighbor (reuses _min_gower_dist_loo from attacks/mia.py).                    │
│ - Returns (outlier_score Series, is_outlier bool Series) keyed to df.index.                                                                 │
│ - Cat/num split inferred from dtype (float64 → continuous, otherwise → categorical) — no config change needed.                              │
│                                                                                                                                             │
│ 1b. master_experiment_script.py — two new helpers + wiring                                                                                  │
│                                                                                                                                             │
│ _build_row_score_path(config, run_id, is_training)                                                                                          │
│ Returns unique path mirroring the data directory hierarchy:                                                                                 │
│ {data_dir}/{sdg_dir}/row_scores/{attack_method}__{qi}__{train_or_holdout}_run{run_id}.csv                                                   │
│ e.g. .../size_10000/sample_01/MST_eps1/row_scores/RandomForest__QI1__train_run0.csv                                                         │
│ Namespaced by (sample, SDG+params, attack, QI, train/holdout, run_id). Never overwrites sibling runs.                                       │
│                                                                                                                                             │
│ _save_row_scores(config, targets, reconstructed, hidden_features, qi, dataset_type, run_id, is_training)                                    │
│ 1. Calls calculate_row_level_scores() → row score DataFrame                                                                                 │
│ 2. Optionally calls compute_outlier_scores() if row_level_analysis.compute_outliers: true                                                   │
│ 3. Builds output DataFrame: [qi_cols | true_{feat}... | recon_{feat}... | RA_row_{feat}... | RA_row_mean | outlier_score | is_outlier |     │
│ is_training | run_id]                                                                                                                       │
│ 4. Saves CSV to path from _build_row_score_path()                                                                                           │
│ 5. Logs to WandB:                                                                                                                           │
│   - Summary scalars: RA_row_p10, RA_row_p50, RA_row_p90 (percentiles of RA_row_mean)                                                        │
│   - If outliers enabled: RA_row_outlier_mean, RA_row_non_outlier_mean (score gap)                                                           │
│   - Histogram: wandb.Histogram(row_scores['RA_row_mean'].values) — distribution visible in WandB UI                                         │
│   - Config key: CSV path stored in wandb.config so it's always traceable                                                                    │
│                                                                                                                                             │
│ Wiring into run_single_experiment():                                                                                                        │
│ - After computing reconstructed_train, check config.get('row_level_analysis', {}).get('enabled', False)                                     │
│ - If enabled: call _save_row_scores(..., is_training=True)                                                                                  │
│ - If memorization test is also enabled and holdout is not None: call _save_row_scores(..., is_training=False) on holdout                    │
│                                                                                                                                             │
│ 1c. New config section (in YAML):                                                                                                           │
│                                                                                                                                             │
│ row_level_analysis:                                                                                                                         │
│   enabled: false                                                                                                                            │
│   compute_outliers: false                                                                                                                   │
│   outlier_method: "isolation_forest"   # or "gower_knn"                                                                                     │
│   outlier_percentile: 90                                                                                                                    │
│                                                                                                                                             │

│ Feature 2: MIA vs RA-as-MIA Comparison                                                                                                      │
│                                                                                                                                             │
│ 2a. attacks/mia.py — add ra_as_mia + raw score returns                                                                                      │
│                                                                                                                                             │
│ ra_as_mia(attack_fn, cfg, synth_df, train_df, holdout_df, qi, hidden_features, n_targets=500, seed=42)                                      │
│ - Samples n_each = min(n_targets, len(train), len(holdout)) records from each                                                               │
│ - Concatenates train_tgts (label=1) + holdout_tgts (label=0) → all_targets                                                                  │
│ - Calls attack_fn(cfg, synth_df, all_targets, qi, hidden_features) → reconstructed                                                          │
│ - Computes per-row reconstruction accuracy inline (no import from scoring.py to avoid path issues):                                         │
│   - Categorical: rarity-weighted row score (same formula as Feature 1)                                                                      │
│   - Continuous: scores = -mean_normalized_error (negate so higher = better reconstruction = higher membership signal)                       │
│ - Calls _mia_metrics(scores, labels) and re-prefixes keys as RA_as_MIA_*                                                                    │
│ - Returns (metrics_dict, scores_array, labels_array, all_targets_df)                                                                        │
│                                                                                                                                             │
│ Add return_raw=False to synth_distance_mia and nndr_mia:                                                                                    │
│ When True, returns (metrics_dict, scores_array, labels_array, all_targets_df) in addition to normal metrics. Backwards-compatible (default  │
│ unchanged).                                                                                                                                 │
│                                                                                                                                             │
│ Do NOT add ra_as_mia to MIA_REGISTRY — it has a different signature (needs attack_fn, qi, hidden_features) and is only used directly by the │
│  comparison script.                                                                                                                         │
│                                                                                                                                             │
│ 2b. attacks/__init__.py — export ra_as_mia                                                                                                  │
│                                                                                                                                             │
│ Add from .mia import ra_as_mia and include in __all__.                                                                                      │

│                                                                                                                                             │
│ run_comparison(sdg_method, sdg_params) — core function:                                                                                     │
│ 1. Build cfg dict (same pattern as make_config() in other scripts)                                                                          │
│ 2. Load train, synth, holdout, meta via load_mia_data(cfg) (already loads all 4)                                                            │
│ 3. Sample aligned targets: same n_each and seed for all methods → ensures record alignment                                                  │
│ 4. Run each MIA_METHODS with return_raw=True → (metrics, scores, labels, targets)                                                           │
│ 5. Run ra_as_mia(attack_fn, cfg, ...) → (metrics, ra_scores, labels, targets)                                                               │
│ 6. Compute outlier scores on the shared all_targets DataFrame (QI cols, QI-space isolation forest)                                          │
│ 7. Call _quadrant_analysis() for each MIA method vs RA-as-MIA                                                                               │
│ 8. Call _outlier_intersection() for all methods                                                                                             │
│ 9. Print rich terminal report                                                                                                               │
│ 10. wandb.log(all_metrics)                                                                                                                  │
│                                                                                                                                             │
│ _quadrant_analysis(mia_scores, ra_scores, labels, mia_name):                                                                                │
│ - Uses median of full sample (train + holdout) as threshold for each score                                                                  │
│ - Reports counts in 4 quadrants for training records only (labels==1):                                                                      │
│   - Both high / MIA-only high / RA-only high / Both low                                                                                     │
│ - Computes Spearman correlation between MIA scores and RA-as-MIA scores for training records                                                │
│                                                                                                                                             │
│ _outlier_intersection(score_dict, outlier_flags, top_k_pct):                                                                                │
│ - For each method's scores: find top-k% records, report fraction that are QI-space outliers                                                 │
│ - Reports baseline (overall outlier rate) for comparison                                                                                    │

IF MIA/RA comparison targets should just be full train + full holdout:
 - _setup in mia.py: n_targets=None → skips .sample(), uses full train and holdout DataFrames as-is. labels length now derived from actual    
  DataFrame lengths rather than hardcoded n_each, so it's correct in both cases.
  - ra_as_mia in mia.py: same None sentinel, same pattern.                                                                                     
  - compare_mia_ra.py: N_TARGETS = None is now the default (use everything).                                                                   
  - example_cfg.yaml: notes that ~ (YAML null) means "all rows".

  One thing to be aware of: if train and holdout are different sizes (e.g. 10k vs 9.8k), the resulting target set will be imbalanced.
  roc_auc_score handles this fine, but MIA_advantage and MIA_balanced_acc are also unaffected since they operate on the ROC curve. So using the
   full sets is safe even if sizes differ slightly.
